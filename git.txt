1. git init==> inicia un repositorio local, una sola vez (copia todo lo que tengo en el directorio donde lo inicio)
2. U==> significa untracked. Sin seguimiento: no estan guardados en el repositorio local
3. Git add==> agrega todos los datos al repositorio local
- git add index.html ==>agregar el archivo al repositorio local, agregando el nombre del archivo
- git add . ==> agrega todo el directorio donde estamos trabajando
4. A ==> archivo agregado al repositorio local
5. M ==> el archivo fue modificado, pero no guardado 
6. git commit ==> Embalar el paquete de archivos para enviar
- m ==> comentar o describir la tarea realizada
Ej: git commit -m "Saludos"
7. Si tocamos algo, volvemos a iniciar, menos el numero 1 (ejecutar git init)
8. Git status ==> consulta el estado de Git
9. Git no guarda directorios(carpetas) vacios por default
10. git remote -v ==> me muestra donde estan guardados los codigos en remoto
11. git log ==> me muestra los commits hechos y quien los hizo
ORDEN: Git add . , git commit , git push
12. Creamos un repo en git Hub.
13. Agregamos la url del repo remoto al git local ==> git remote add origin http: //elreporemoto.git
14. Enviamos el repo con git push origin master.
15. Git log one line ==> me muestra los commits resumidos en una sola linea
